## 一、项目概述

### 1.1 项目名称
物采通 — 废旧物资回收行业多商户 SaaS 交易平台

### 1.2 项目定位
面向废旧物资收购行业，为 N 个收购商提供各自独立品牌的线上收购站点，完成采购→运输→过磅→验收→签章→支付→开票全链路业务。

### 1.3 平台定位
- 纯技术服务平台
- 提供身份识别、电子签章、反向开票等技术接口
- 维护品种字典和商户准入
- 不核验数据真实性、不介入纠纷、不处理经纪人佣金

## 二、角色体系

### 2.1 角色总览

| 角色 | 主体类型 | 谁准入 | 业务职责 |
|------|----------|--------|----------|
| 平台管理员 W | W方内部 | 系统预设 | 品种字典维护、C/B入驻审批、C-B关系绑定 |
| 终端收购商 C | 企业法人 | W审批 | 发布采购公告、直采、反向开票、管理仓管员 |
| 贸易收购商 B | 企业法人 | W审批 | 供应商准入、转挂销售、反向开票、管理仓管员 |
| 供应商 A | 个人/个体户 | C或B自行准入 | 认售、交货、签章、收款 |
| 经纪人 | 个人/企业 | C或B邀请 | 协助采购、代填供应商、安排运输、收劳务费（线下） |
| 司机 | 个人 | C或B或经纪人邀请 | 执行运输、GPS打卡、上传磅单照片 |
| 仓管员 | 个人 | C或B管理 | 仓库入库验收、电子签章 |

### 2.2 W的管控范围

| 管控内容 | 说明 |
|----------|------|
| 品种字典 | 上架/编辑/停用商品大类 |
| 终端收购商C入驻 | 申请→审批→激活 |
| 贸易收购商B入驻 | 申请→审批→激活 |
| C-B交易关系绑定 | 按品种大类绑定，B的品种必须 ⊆ C的品种 |

### 2.3 商户自行管理范围

| 角色 | 管理内容 |
|------|----------|
| C | 供应商A准入、经纪人邀请、司机邀请、仓管员管理 |
| B | 供应商A准入、经纪人邀请、司机邀请、仓管员管理 |

## 三、核心业务流程

### 3.1 模式A：直采（C ← A）

```
步骤1   C发布采购公告
           ↓
步骤2   A认售
           ↓
步骤3   C确认认售，生成订单（DIRECT）
           ↓
步骤4   派车（或卖方送货跳过）
           ↓
步骤5   司机到达供应商处，GPS打卡
           ↓
步骤6   源头过磅，司机上传磅单照片
           ↓
步骤7   司机装车出发
           ↓
步骤8   司机到达C仓库，GPS打卡
           ↓
步骤9   仓库复磅，上传磅单照片
           ↓
步骤10  C仓管员入库验收，电子签章
           ↓
步骤11  系统生成合同，C和A双方电子签章
           ↓
步骤12  C通过平台支付货款
           ↓
步骤13  完成
```

### 3.2 模式B：带贸易商（C ← B ← A）

```
C发布采购公告
    ↓
B认售
    ↓
C确认B的认售，生成C←B主订单（TRADE）
    ↓
B代填供应商A信息，生成B←A子订单（SUB）
    ↓
[B←A子订单独立流转，物理流向：A → B仓库]
  司机接单 → GPS打卡 → 源头过磅（司机上传）
  → B仓管员验收签章 → 合同签章 → B支付A
    ↓
[C←B主订单流转，物理流向：B → C仓库]
  派车 → GPS打卡 → C仓管员验收签章
  → 合同签章 → C支付B
    ↓
B反向开票给A
    ↓
B正向开票给C
    ↓
完成
```

### 3.3 经纪人辅助（可附加到模式A或模式B）

```
C或B创建订单时选择"经纪人协助"
    ↓
经纪人接受任务
    ↓
经纪人代填供应商信息
    ↓
系统生成对应订单（DIRECT 或 SUB）
    ↓
经纪人协助安排运输、过磅
    ↓
后续流程同模式A
    ↓
劳务费线下结算，平台不涉及
```

### 3.4 物流节点说明

| 场景 | 跳过节点 |
|------|----------|
| 卖方自行送货（无司机） | 派车→源头打卡→源头过磅→运输中 |
| 无仓管员（司机兼签收） | 入库中→入库完成单独仓管签章 |
| 贸易商链路子订单 | 视B是否有司机决定是否派车 |

## 四、订单状态机

### 4.1 状态枚举

| 状态 | 说明 |
|------|------|
| DRAFT | 草稿 |
| COMMITTED | 已确认认售 |
| DISPATCHED | 已派车 |
| ARRIVED_SOURCE | 司机到达供应商处 |
| SOURCE_WEIGHED | 源头过磅完成 |
| IN_TRANSIT | 运输中 |
| ARRIVED_WAREHOUSE | 司机到达仓库 |
| WAREHOUSE_WEIGHED | 仓库复磅完成 |
| WAREHOUSING | 入库中 |
| WAREHOUSED | 入库完成 |
| CONTRACT_PENDING | 待签章 |
| CONTRACTED | 已签章 |
| PAYING | 付款中 |
| PAID | 已付款（终态之一） |
| COMPLETED | 已完成（终态） |
| CANCELLED | 已取消（终态） |

### 4.2 合法状态转换

```python
transitions = {
    'DRAFT':               ['COMMITTED', 'CANCELLED'],
    'COMMITTED':           ['DISPATCHED', 'ARRIVED_WAREHOUSE', 'CANCELLED'],
    'DISPATCHED':          ['ARRIVED_SOURCE', 'CANCELLED'],
    'ARRIVED_SOURCE':      ['SOURCE_WEIGHED'],
    'SOURCE_WEIGHED':      ['IN_TRANSIT'],
    'IN_TRANSIT':          ['ARRIVED_WAREHOUSE'],
    'ARRIVED_WAREHOUSE':   ['WAREHOUSE_WEIGHED', 'CONTRACT_PENDING'],
    'WAREHOUSE_WEIGHED':   ['WAREHOUSING', 'CONTRACT_PENDING'],
    'WAREHOUSING':         ['WAREHOUSED'],
    'WAREHOUSED':          ['CONTRACT_PENDING'],
    'CONTRACT_PENDING':    ['CONTRACTED'],
    'CONTRACTED':          ['PAYING'],
    'PAYING':              ['PAID'],
    'PAID':                ['COMPLETED'],
}
```
